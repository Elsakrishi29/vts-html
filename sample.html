IAM

What is IAM?

IAM (Identity and Access Management) is used to manage users, roles, and permissions in AWS.

What is an IAM User?

An IAM User is a person or application that can log in to AWS using username/password or access keys.

IAM Policy vs IAM Role

IAM policies define permissions, and IAM roles are used to assign those permissions securely to AWS services or users. IAM Policy Example: Allow EC2 read access, deny S3 delete access.

IAM Role is mainly used by AWS services (EC2, ECS, Lambda) to access other AWS resources securely without access keys. IAM Role Example:EC2 uses a role to access S3.

What is an inline policy?

An inline policy is a policy directly attached to one user or role and cannot be reused.

If AWS keys are leaked, what would you do?

Immediately disable the leaked keys, rotate credentials, and check logs.

Cross Account Access

Cross-account access allows one AWS account to access resources in another account using IAM roles.

S3 Bucket

Is there any limit for S3 buckets?

Yes, S3 bucket limit is there. By default, we can create 10,000 buckets per AWS account. If we need more, we can request a quota increase through the Service Quotas console.

Is this limit per region or per account?

It is per AWS account, not per region.

What is Amazon S3?

Amazon S3 (Simple Storage Service) is an object storage service in AWS.
It is used to store and retrieve any type of data like files, images, videos, logs, and backups.

What are S3 buckets?

S3 bucket is a container used to store objects like files, images, and videos in AWS.

What is S3 Versioning?

S3 Versioning keeps multiple versions of the same object to protect against accidental deletion or overwrite.

What is Auto scaling?

Auto Scaling automatically increases or decreases the number of servers based on application traffic.
If traffic increases, it adds more instances.
If traffic decreases, it removes extra instances.
This helps maintain performance and reduce cost.

What is Infrastructure as Code?

Infrastructure as Code means managing and creating infrastructure using code instead of manual configuration.

For example, servers, networks, and storage can be created automatically using tools like Terraform or AWS CloudFormation.

What is Terraform?

Terraform is an Infrastructure as Code tool used to create and manage cloud resources automatically.

What are Terraform modules?

Terraform modules are reusable blocks of code in Terraform.

They help organize infrastructure code and reuse it in different projects.

For example, a module can be created to deploy an EC2 instance and reused multiple times.

Kubernetes

Kubernetes vs Docker

Docker is used to build and run containers.

Kubernetes is used to manage, scale, and deploy containers.

What is Kubernetes?

Kubernetes is a container orchestration tool used to deploy, manage, scale, and monitor containerized applications.

It provides self-healing, auto-scaling, and automatic container management, which is why it is called an orchestration tool.

What is Pod

A Pod is the smallest deployable unit in Kubernetes that runs one or more containers.
Pods in Kubernetes are ephemeral, meaning they can be created and deleted anytime.

What is a Node?

A Node is a machine (virtual or physical) where Pods run in Kubernetes.
Kubernetes has two types of nodes:
* Control Plane (Master Node) => Manages and controls the cluster and worker nodes.
* Worker Node => Runs the Pods and containers.
Master node is called the control plane. because its manged the worker node.

Difference between Pod and Container

A container is a single running instance of an application.

A Pod is the smallest unit in Kubernetes and it can contain one or more containers.

Containers inside a Pod share the same network and storage.

What is Deployment

Deployment in Kubernetes is used to manage and update Pods.

It ensures the required number of Pods are always running and helps with updates and scaling.

What is Service

A Service in Kubernetes provides a stable network endpoint to access Pods.

Even if Pods restart or change IP addresses, the Service allows applications to communicate reliably.

What is Ingress

Ingress in Kubernetes manages external access to services.

It allows users to access applications using domain names or HTTP/HTTPS routing.

What is CI/CD? Explain the pipeline flow.

CI/CD stands for Continuous Integration and Continuous Deployment.
It is used to automatically build, test, and deploy applications when developers push code to a repository.

Pipeline flow:

  • Developer pushes code to repository
  • CI pipeline starts automatically
  • Application is built
  • Tests are executed
  • If tests pass, the application is deployed

How do you create a pipeline in Jenkins?

I have basic knowledge of creating pipelines in Jenkins.
In Jenkins, we create a pipeline job and define the build steps in a Jenkinsfile.
The pipeline usually includes stages like build, test, and deployment.

Difference between Freestyle Job and Pipeline Job

Freestyle Job
Simple job configuration
Steps added manually in Jenkins UI Pipeline Job
Uses code (Jenkinsfile)
Supports multiple stages like build, test, deploy
Easier to maintain in version control

What is Jenkinsfile?

Jenkinsfile is a configuration file used to define the CI/CD pipeline stages.
It contains instructions for build, test, and deployment steps.

pipeline {
agent any
stages {
stage('Build') {
steps {
echo 'Building application'
}
}
}
}
